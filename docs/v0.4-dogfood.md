## Dogfood Shape

The run used:

- one Millrace daemon session named `v04-daemon`;
- two agent-role PTY sessions named `v04-agent-a` and `v04-agent-b`;
- an Agent Cockpit opened once, detached, then reopened with the same UI id;
- distinct attention states on both agents;
- safe input delivery to `v04-agent-a`;
- `millmux context export` as the handoff artifact.

The two agent-role sessions used small bash loops instead of invoking an LLM
operator. That kept the proof deterministic while still exercising Millmux PTY
session, input, attention, cockpit, and handoff behavior.

## Command Pattern

```bash
cd /mnt/f/_Millrace/mac-handoff/dev/millmux
cargo build --workspace

WORKSPACE=/mnt/f/_Millrace/mac-handoff/test
STATE=/tmp/millmux-v0.4-dogfood-20260709T151641Z-state
EVIDENCE=/mnt/f/_Millrace/mac-handoff/test/millmux-v0.4-dogfood-20260709T151641Z/evidence
MM=/mnt/f/_Millrace/mac-handoff/dev/millmux/target/debug/millmux
export MILLMUX_STATE_DIR="$STATE"
export MILLMUX_HOST_BIN=/mnt/f/_Millrace/mac-handoff/dev/millmux/target/debug/millrace-sessiond

"$MM" start --name v04-daemon --workspace "$WORKSPACE" --cwd "$WORKSPACE" \
  --role millrace-daemon --monitor basic --json -- \
  millrace run daemon --workspace "$WORKSPACE" --monitor basic

"$MM" agent start --name v04-agent-a --workspace "$WORKSPACE" --cwd "$WORKSPACE" \
  --json -- /usr/bin/bash -lc 'printf "agent-a-ready\n"; while IFS= read -r line; do printf "agent-a-received:%s\n" "$line"; done'

"$MM" agent start --name v04-agent-b --workspace "$WORKSPACE" --cwd "$WORKSPACE" \
  --json -- /usr/bin/bash -lc 'printf "agent-b-ready\n"; while IFS= read -r line; do printf "agent-b-received:%s\n" "$line"; done'

"$MM" cockpit --workspace "$WORKSPACE" --monitor basic \
  --ui 0011aac1-41fe-4d9d-9ed0-168ce550326e --no-start --once \
  --agent-argv -- /usr/bin/bash -lc '<same agent A script>'

"$MM" attention mark v04-agent-a --kind unread --severity warning \
  --source operator --message "detached attention while cockpit closed" \
  --dedupe-key v04-detached --json

"$MM" cockpit --workspace "$WORKSPACE" --monitor basic \
  --ui 0011aac1-41fe-4d9d-9ed0-168ce550326e --no-start --once \
  --agent-argv -- /usr/bin/bash -lc '<same agent A script>'

"$MM" input send --text $'safe Batch 5 prompt for agent A\n' v04-agent-a --json

"$MM" context export --ui 0011aac1-41fe-4d9d-9ed0-168ce550326e --json \
  --objective "Batch 5 dogfood recovery" \
  --note "WSL dogfood run 20260709T151641Z" \
  --source-session v04-agent-a \
  --destination-session v04-agent-b
```

Raw PTY logs, replay rings, and full screen JSON are intentionally referenced by
path and checksum instead of pasted into this document. They can include prompts,
workspace paths, or other operator-local details.

`context-export.json` includes local `/tmp/...` state paths as source
references, because Millmux keeps Unix sockets and raw session artifacts under
the active state root. The portable dogfood ledger is the `evidence/` directory
above plus `artifact-sha256.txt`; the `/tmp/...` references are local replay
pointers, not a standalone release artifact.

## What This Proves

- WSL/Linux can start and observe a Millrace daemon under Millmux.
- Two agent-role PTY sessions can run beside the daemon.
- Cockpit can write UI context, detach, reopen by UI id, and keep hosted
  sessions alive.
- Attention accumulated while cockpit was detached is visible in the exported
  handoff bundle.
- Session-targeted input delivery reaches the hosted agent PTY.
- `millmux context export` now produces a bounded handoff bundle with source
  attribution, session summaries, attention, logs, events, screen summaries,
  and artifact references.

## Limits

- This run used deterministic bash-backed agent-role sessions, not live LLM
  agents.
- The interactive scroll/search/copy/paste behavior is covered by automated
  TUI tests and prior cockpit evidence; this WSL dogfood used non-interactive
  `--once` cockpit snapshots for repeatability.
- macOS Terminal.app and SSH terminal matrices were not rerun in this handoff.
